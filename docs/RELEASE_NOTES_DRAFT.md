# YangMind 版本更新草稿（对话记录，待发公告）

> 从 2026-07-22 本轮对话起累计。正式对外文案可再润色后发到「公告栏」。

## 进行中 / 本轮（优化）— 已实现，待部署后发公告

### 登录与注册
- 支持使用**邮箱或昵称**登录（不再强制登录框必须是邮箱格式）
- 注册时**昵称不可重复**（大小写不敏感；启动时自动处理历史重名并建唯一索引）

### 管理端体验
- 优化「用户数据」首屏：用户列表改为批量查询，避免逐用户 N+1；统计与列表并行请求
- 管理端各页切页增加转圈加载，避免「暂无…」挤在左侧小框一闪而过
- 「用户数据」首屏在数据未就绪时显示大号「正在加载用户数据…」转圈，避免干等空白/破折号
- 空列表改为全宽提示
- 管理端模态统一 Portal 到 `body`，遮罩覆盖整视口（含侧栏与顶栏）
- 公告弹层改为视口垂直水平居中
- 页面切换入场动画改为仅透明度，减少 `fixed` 遮罩被截断的风险

### 管理员演示模式
- 总管/子管均可进入「演示模式」，完整交互问卷与猎鹿博弈（可重复）
- 演示数据仅存服务端内存，**不写入正式库、不进 CSV**
- 人机对手每轮约 60% 偏向一侧，长期 A/B 近似均匀
- 顶栏提示「演示 · 数据不保存」，支持一键重置演示

### 管理端侧栏重构（面向运营同学）
- 侧栏改为「分组标题 + 下级菜单」
- 总管：实验运营（参与者数据 / 博弈实验）→ 拉人与协作（邀请码 / 子管理员）→ 站点配置（参与端页面 / 文案与公告）→ 账号
- 子管：我的团队（我的参与者 / 我的邀请码）→ 账号
- 去掉含糊的「注册与登录」；邀请码与子管理员分开展示
- 侧栏改用 Lucide SVG 图标（人群 / 烧瓶 / 钥匙 / 协作账号 / 列表 / 喇叭 / 剪贴板 / 用户等）
- 「操作记录」从邀请码页拆出，独立成「系统 → 操作记录」

### 调试账号 sudo
- 环境变量：`ENABLE_SUDO` / `SUDO_PASSWORD` / `SUDO_NICKNAME=sudo`（可用昵称登录）
- `role=sudo`，权限覆盖总管；仅此账号右上角有「总管 | 子管 | 参与者」视角切换
- 发出的邀请码与经该码注册的用户带 `is_debug`；正式统计/排行榜/总管导出默认排除
- sudo 导出含调试数据，并带 `is_debug` 字段，便于测表头

### BFI 答题区改版
- 去掉大留白 hero，改为紧凑顶栏（进度条 / 已答数 / 自动保存）
- 题目改为量表矩阵（统一 1–5 表头）；桌面右侧进度栏（分组进度 + 定位未答题）
- 手机端改为单栏，进度栏置顶

### 博弈结束页人格反馈
- 真人匹配两场景全部结束后，结束页直接展示人格倾向文案与五维得分（1–5 + 进度条）
- 仍遵循「问卷提交后延迟解锁」：仅全部必做场景完成后才返回人格数据
- 演示模式同样在对局结束页展示演示人格结果
